# Security Policy

## Reporting a Vulnerability

**Do NOT open a public GitHub issue for security vulnerabilities.**

Report security vulnerabilities through the **Immunefi bug bounty programme** (link published at mainnet launch), or by emailing **security@nvxo.io** with:

1. Description of the vulnerability
2. Steps to reproduce
3. Potential impact assessment
4. Any proof-of-concept code (optional)

We will acknowledge receipt within 24 hours and provide a detailed response within 72 hours.

## Scope

In scope for bug bounty:
- All deployed NVXO smart contracts on mainnet
- NVXO Chain bridge contracts
- DEX router and pool contracts
- Staking and governance contracts

Out of scope:
- Frontend / UI issues
- Theoretical attacks without proof-of-concept
- Issues already known / in audit queue

## Responsible Disclosure

We follow a 90-day responsible disclosure policy. After a fix is deployed, we will credit the researcher (if desired) in the relevant audit report addendum.
